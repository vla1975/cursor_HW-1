Перелік основних команд

  • git config - дозволяє встановити ім'я користувача, при налаштуванні Git
  • git config - дозволяє встановити електронну пошту користувача, при налаштуванні Git
  • git init - дозволяє перетворити звичайну директорію на новий репозиторій Git
  • git status - показує різні стани файлів у робочій директорії та індексі. Які файли змінені, проте не в індексі, а які індексовані, проте досі не збережені в коміті
  • git add - додає вміст з робочої директорії до індексу (чи області додавання) для наступного коміту
  • git commit - бере вміст всіх файлів, які індексовані командою git add, та записує новий сталий відбиток до бази даних, а потім пересуває вказівник поточної гілки до нього
  • git log - використовується, щоб показати історію проекту, починаючи з останнього відбитку коміту у зворотному порядку
  • git checkout - використовується для переключення гілок та отримання вмісту до робочої директорії
  • git branch - дозволяє виводити список існуючих гілок, створювати нову гілку, вилучати гілки та перейменовувати їх
  • git push - використовується для надсилання змін до віддаленого сховища
  • git pull - використовується для відправки змін зі заданого віддаленого сховища. При виклику цієї команди зразу відбудеться спроба злити циз змін з поточною гілкою
  • git merge - використовується для зливання однієї чи більше гілок до поточної гілки. Вона потім пересуває поточну гілку до результату злиття
  • git clone - створює нову директорію та додає віддалене сховище з URL, яке ми вказали при виклику команди

Файл .gitignore - це текстовий файл, який зберігає дані про файли та директорії, які не будуть відправлені на віддалений сервер (будуть проігноровані)

Advanced

  • git revert - створює новий коміт, який застосовує точну протилежність попереднім комітам, по суті скасовуючи чи вивертаючи ї
  • git reset - вилучає файли з індексу
  • git rebase - використовується для застосування комітів в іншій гілці
  • git cherry-pick - використовується для того, щоб взяти впроваджені ввикористовується для тимчасового збереження роботи поза комітом, щоб очистити робочу директорії без необхідності створювати коміт з незавершеною роботою в гілці одному коміті Git зміни, та спробувати застосувати їх як новий коміт на поточній гілці
  • git stash - використовується для тимчасового збереження роботи поза комітом, щоб очистити робочу директорії без необхідності створювати коміт з незавершеною роботою в гілці
